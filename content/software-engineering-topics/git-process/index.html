In this activity, you will practice your Git knowledge. You will follow a set of steps and will take screenshots of your work as a proof of completing the tasks. You will include all your screenshots in a word file and submit one file.

Submission format - You will work individually on this task, however, for one of the steps (reviewing your pull request) you will need to add a friend to your repository and ask them to do this step for you.

Estimated time: 1 hour

Accounts and installations

  1. We will use Github in this activity. If you do not have a Github account, please create an account at https://github.com/ before you start.
  2. Then, make sure that Git is installed on your computer. You can check the version of your git with git --version. If it is not installed, please install it from https://git-scm.com/downloads

Create a repository and set the permissions

  1. Create a private repository in Github. Name your repository as ‘exercise-XX', where XX is your initials, e.g., John Doe will create exercise-JD repository. Select to create a readme file from the UI.
  1. Give access to your friend who will play the reviewer role for you in steps 17-18 of this activity.

Global and local configs

  1. Now, check the list of the global configs on your computer. Make sure that your Git username and email address are set correctly, before cloning your remote repository to your computer. Take a snapshot of your global configs.
  1. Clone your repository on your computer and go to your cloned directly.
  1. Now, list all files (including hidden files) in this directory. You should see a hidden folder called ‘.git' which includes all your git related files and information. Take a screenshot of all files (including hidden files) under the .git folder.
  1. Check each file under .git and guess what each file is for. Search and learn about the ones that you do not know. There is no need to submit your answer for this step.

Issue and project settings

  1. In your github file create a project (automated kanban with reviews). Give it a name of your choice. Go to the project and remove the auto generated cards from your kanban board.
  1. Create an issue of ‘Questions in exercise 1 need to be completed' in your repository and assign it to yourself and to your project, and select a label for it. Check your kanban and make sure that the issue is listed in your project. Remember the ID of the issue you created (We will need this number soon!).
  1. Check the existing branches in your local repository. You must have a default branch, such as ‘master' branch. If ‘develop' branch does not exist, create a branch and name it ‘develop'. Then, push your branch to the remote repository. Take a screenshot that shows you have two branches in your local and also in your remote repositories.
  1. Your task ‘Questions in exercise 1 need to be completed' is to write answers for two questions. Question 1 - What is the .gitignore file and how can you create it? Question 2 - Search and find a feature or a useful tip about Git that we have not touched in class and describe it. You will complete these two questions in two separate commands. For example, answer the first question and commit it, and then answer the second question. You can write your answers in a .md or .html file that you create. Let's start.
  1. Move your task to in-progress and start your work. Create a local branch for your issue. For example: git checkout -b issue-ID/exercise-answers, where ID is your issue number. You can find it your issue tab or in your kanban.
  1. In this newly created branch, create a file and write your answer for the first question. Use add and commit commands with proper message to commit your work, e.g., git add answers.md, and then git commit -m ‘answer to first question is completed'.
  1. Now, add your answer to the second question and commit it with a proper message. Check the logs using git log and take a screenshot.
  1. Now, push your issue branch to your remote repository, e.g., git push origin issue-ID/exercise-answers
  1. Go to your remote repository and create a pull request to merge your changes to the develop branch. When creating the pull request, assign it to your project, and select the reviewer to be your friend (who already has accepted your invite to join your repository). For the description use "resolves #ID", where ID is the ID of the issue that you have completed.
  1. Go to your Kanban and make sure that the issue is in the proper column, waiting for a review. Wait until the reviewer approves your changes. After the review, your issue should be in the reviewed column.
  1. After your pull request is approved, it is time to merge it. Go to the pull request tab and squash and merge the commits. Merge commit comments and add one single readable comment, and then do the merge. After this step, your issue must be in the done column in your kanban. Take a screenshot of your kanban.
  1. Check that the codes in your develop branch includes your changes, and then delete your branch from the remote repository, and also from your local repository, e.g., with git branch -d <name of the branch>. Note that before deleting your local branch, you should first switch to a different branch, e.g., develop branch. Take a screenshot showing that your local and remote repositories both have only two branches.
  1. Pull the the newly added files to the develop branch from the remote repository to update your local develop branch. Now, both your remote and local repositories should be up to date.
  1. Finally, take a screenshot of the commit history for your develop branch. It should only contain one commit for the both changes (commits) that you merged.
  1. You finished your task. Include all your screenshots in a word file and submit one file.
