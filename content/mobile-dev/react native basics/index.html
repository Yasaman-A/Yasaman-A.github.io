In this activity, you will practice your react native knowledge by creating a single screen mobile application. You will create your mobile application and store your code in your Github repository and also on Expo snack.

Submission format - You will work individually on this task and 1) include your code in your github repository. You will invite your instructor to your github repository to review your work. 2) And also, you will create an Expo snack and add your code. You will submit the link of your snack in BlackBoard.

Estimated time: 2 hours

Submission deadline: Oct. 21

Accounts and installations

  1. You will use Github in this activity. Create a repository in your Github account.

Readings and practices before you start

  1. Make sure to read the following sections from React Native documentation: https://reactnative.dev/docs/getting-started: 1) Introduction, 2) Core and native components, 3) React fundamentals.
  1. Make sure to review the code samples we completed in class.
  1. You will create a single screen (one page) GPA calculator that shows the name of 3 courses, e.g., course 1, course 2, and course 3. The user can enter a grade out of 100 for each course. When the user enters the grade, the letter grade will be calculated and shown on the page for each course.
  1. When the user clicks on the calculate button, the application will calculate the average of the grades entered as the GPA (assuming that all grades have the same weight and credit). When the average is calculated, it will be shown on the screen.
  1. Design your code using two components: Course component that represents a course and a calculate component that does your calculations.
  1. Please test the application with different possible user inputs. Your application should not crash with any input that the user is allowed to enter in the application. You can test it on your phone, or emulators/simulators, or you may use Expo Snack (https://snack.expo.io/) to test the output of your application online.
  1. A sample design is provided below. This design is based on the components of react-native-paper: https://callstack.github.io/react-native-paper/. You may choose to use a different library or style your own components. Your final design does not have to look exactly the same as the provided sample.

  1. After you finish writing the code, use Eslinter locally to check the style of your code, and fix the identified style issues.
  1. Push your changes to your Github repository. You may use any branch for this exercise. Make sure to have a workflow that checks your code style and the workflow runs successfully when you push your code.
  1. Create an Expo Snack account for yourself and create files and copy-paste your code to your Snack to see the output. Note that when using Snack you may need to write the full name of the imported files, e.g., instead of import Game from './Game'; you will need to use import Game from './Game.jsx';
  1. To complete your task, 1) share your github with your instructor (Github username: Yasaman-A) for review and also 2) submit the link of your Expo Snack in Blackboard.
